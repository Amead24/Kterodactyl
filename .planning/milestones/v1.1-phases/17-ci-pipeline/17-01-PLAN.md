Create `.github/workflows/ci.yml` with the following structure. Use the existing workflow files and RESEARCH.md patterns as reference.

**Trigger:** `on: push: branches: [main]` and `on: pull_request:`

**Job 1 — lint** (no dependencies):
- runs-on: ubuntu-latest
- actions/checkout@v4
- actions/setup-go@v5 with go-version-file: go.mod
- golangci/golangci-lint-action@v8 with version: v2.7.2

**Job 2 — unit-test** (needs: [lint]):
- runs-on: ubuntu-latest
- actions/checkout@v4
- actions/setup-go@v5 with go-version-file: go.mod
- run: make test

**Job 3 — integration-test** (needs: [lint]):
- runs-on: ubuntu-latest
- actions/checkout@v4
- actions/setup-go@v5 with go-version-file: go.mod
- run: make test-integration

**Job 4 — e2e-test** (needs: [unit-test, integration-test]):
- runs-on: ubuntu-latest
- jlumbroso/free-disk-space@main with: tool-cache: false, android: true, dotnet: true, haskell: true, large-packages: true, docker-images: false, swap-storage: false
- actions/checkout@v4
- actions/setup-go@v5 with go-version-file: go.mod
- Install kind: curl -Lo ./kind https://kind.sigs.k8s.io/dl/latest/kind-linux-amd64, chmod +x, sudo mv to /usr/local/bin/kind
- run: make test-e2e-setup
- run: KIND=kind KIND_CLUSTER=kterodactyl-test-e2e go test -tags=e2e ./test/e2e/ -v -ginkgo.v
  (Do NOT use `make test-e2e` because it chains cleanup — we need separate steps for `if: always()`)
- Capture k8s logs (if: failure()): kubectl get pods -A -o wide, kubectl logs -l app.kubernetes.io/name=kterodactyl -n kterodactyl-system --tail=500, kubectl describe pods -n kterodactyl-system — write all to /tmp/k8s-logs/ directory
- Upload k8s logs artifact (if: failure()): actions/upload-artifact@v4, name: e2e-k8s-logs, path: /tmp/k8s-logs/, retention-days: 14
- Cleanup kind cluster (if: always()): kind delete cluster --name kterodactyl-test-e2e

**Job 5 — playwright** (needs: [e2e-test]):
- runs-on: ubuntu-latest
- jlumbroso/free-disk-space@main (same config as e2e-test)
- actions/checkout@v4
- actions/setup-go@v5 with go-version-file: go.mod
- actions/setup-node@v5 with node-version: 'lts/*'
- Install Playwright browsers: working-directory: e2e, run: npm ci && npx playwright install --with-deps chromium
- Install kind (same as e2e-test)
- run: make test-e2e-setup
- Run Playwright tests: working-directory: e2e, run: npx playwright test
- Capture k8s logs (if: failure()): same pattern as e2e-test, write to /tmp/k8s-logs/
- Upload Playwright report (if: ${{ !cancelled() }}): actions/upload-artifact@v4, name: playwright-report, path: e2e/playwright-report/, retention-days: 14
- Upload Playwright test results (if: ${{ !cancelled() }}): actions/upload-artifact@v4, name: playwright-test-results, path: e2e/test-results/, retention-days: 14
- Upload k8s logs (if: failure()): actions/upload-artifact@v4, name: playwright-k8s-logs, path: /tmp/k8s-logs/, retention-days: 14
- Cleanup kind cluster (if: always()): kind delete cluster --name kterodactyl-test-e2e

CRITICAL details:
- Do NOT use `make test-e2e` for the e2e-test job — it chains cleanup in Makefile which is skipped on failure. Use `make test-e2e-setup` + explicit test command + `if: always()` cleanup step.
- Do NOT remove docker-images or tool-cache in free-disk-space (kind needs Docker, setup-go/setup-node need tool-cache).
- Use `if: ${{ !cancelled() }}` for Playwright artifact uploads (not `if: failure()`) so reports upload even on test failure.
- Use `if: failure()` for k8s log capture (only useful on failure).
- Use `if: always()` for kind cleanup steps.
- Artifact names must be unique across jobs (e2e-k8s-logs vs playwright-k8s-logs).
