  <action>
1. Run `go get github.com/santhosh-tekuri/jsonschema/v6@v6.0.2` to add the JSON Schema library.

2. Delete the old flat file `games/minecraft.yaml`.

3. Create `games/minecraft/manifest.yaml` with the full Minecraft reference manifest from the research. Include all fields: name, displayName, image, ports, resources, parameters (EULA, TYPE, DIFFICULTY, MODE, MAX_PLAYERS, MEMORY, MOTD, PVP, SEED, ONLINE_MODE), and a complete parameterSchema section with JSON Schema Draft 2020-12. Every parameter should use `type: string` with appropriate constraints (enum, pattern, const, maxLength, default). Include `title` and `description` on every property for UI metadata. Add `required: [EULA, TYPE]`.

4. Create `games/minecraft/Dockerfile` as a thin wrapper: `FROM itzg/minecraft-server:latest` with documentation comments explaining this is a reference implementation and linking to the itzg docs.

5. Update `internal/manifest/manifest.go`:
   - Add import for `"bytes"`, `"encoding/json"`, and `"github.com/santhosh-tekuri/jsonschema/v6"`.
   - Add `ParameterSchema map[string]interface{}` field to `GameManifest` with `yaml:"-"` tag (unmarshaled via raw type).
   - Add unexported `compiledSchema *jsonschema.Schema` field to `GameManifest`.
   - Add `ParameterSchema map[string]interface{}` field to `rawGameManifest` with `yaml:"parameterSchema"` tag.
   - Update `LoadFromDirectory` to scan subdirectories instead of flat files:
     - Iterate entries; for each directory entry, look for `manifest.yaml` (or `manifest.yml`) inside it.
     - Skip non-directory entries (no backward compat for flat files needed since we are migrating the only game).
     - After parsing the raw manifest, if `raw.ParameterSchema != nil`, compile the schema: marshal to JSON, create `jsonschema.NewCompiler()`, add resource with URL `games/<name>/manifest.yaml#/parameterSchema`, compile. Store compiled schema in the `GameManifest`.
     - Set `ParameterSchema: raw.ParameterSchema` on the GameManifest struct.
   - Keep all existing logic for port conversion, resource parsing, required field validation intact.

6. Create `internal/manifest/validate.go`:
   - Package `manifest`.
   - Import `"fmt"` and `"github.com/santhosh-tekuri/jsonschema/v6"`.
   - Define method `func (m *GameManifest) ValidateParameters(params map[string]string) error`:
     - If `m.compiledSchema == nil`, return nil (no schema = no validation).
     - Convert `map[string]string` to `map[string]interface{}` (each string value stays as string).
     - Call `m.compiledSchema.Validate(instance)`.
     - If validation fails, return `fmt.Errorf("parameter validation failed: %w", err)`.
     - On success, return nil.
   - Define accessor `func (m *GameManifest) HasSchema() bool` returning `m.compiledSchema != nil`.

Do NOT change Parameters from `map[string]string` -- env vars are always strings. The JSON Schema uses `type: string` for all properties.
  </action>
