**RBAC Templates** -- Convert each kustomize RBAC file into a Helm template. Critical rules:
1. ClusterRole/ClusterRoleBinding names use `{{ include "kterodactyl.fullname" . }}-<suffix>` to prevent multi-install collisions (Pitfall 4 from research)
2. Namespaced Role/RoleBinding names also use fullname prefix
3. All ClusterRoleBinding/RoleBinding subjects reference the ServiceAccount via `{{ include "kterodactyl.serviceAccountName" . }}` with namespace `{{ .Release.Namespace }}`
4. All resources get Helm standard labels via `{{ include "kterodactyl.labels" . | nindent 4 }}`
5. RBAC rules stay IDENTICAL to the kustomize source -- do NOT modify any apiGroups, resources, or verbs

Specific files:
- **clusterrole.yaml**: Convert config/rbac/role.yaml. Name: `{{ include "kterodactyl.fullname" . }}-manager-role`. Copy ALL rules verbatim (core resources, game.kterodactyl.io, gateway.networking.k8s.io, networking.k8s.io).
- **clusterrolebinding.yaml**: Convert config/rbac/role_binding.yaml. Name: `{{ include "kterodactyl.fullname" . }}-manager-rolebinding`. roleRef.name must match the ClusterRole name exactly.
- **role-leader-election.yaml**: Convert config/rbac/leader_election_role.yaml. Namespaced Role in `{{ .Release.Namespace }}`. Name: `{{ include "kterodactyl.fullname" . }}-leader-election-role`. Rules: configmaps, leases, events.
- **rolebinding-leader-election.yaml**: Convert config/rbac/leader_election_role_binding.yaml. Name: `{{ include "kterodactyl.fullname" . }}-leader-election-rolebinding`. roleRef.name matches Role.
- **clusterrole-metrics-auth.yaml**: Convert config/rbac/metrics_auth_role.yaml. Name: `{{ include "kterodactyl.fullname" . }}-metrics-auth-role`. Rules: tokenreviews, subjectaccessreviews.
- **clusterrolebinding-metrics-auth.yaml**: Convert config/rbac/metrics_auth_role_binding.yaml. Name: `{{ include "kterodactyl.fullname" . }}-metrics-auth-rolebinding`.
- **clusterrole-metrics-reader.yaml**: Convert config/rbac/metrics_reader_role.yaml. Name: `{{ include "kterodactyl.fullname" . }}-metrics-reader`. Rules: nonResourceURLs /metrics.

**AdminConfig ConfigMap** (configmap-admin.yaml):
CRITICAL: The ConfigMap name MUST be hardcoded as `kterodactyl-admin-config` -- NOT using the fullname helper. The Go source at internal/controller/gameserver_controller.go:67 hardcodes this name. Using a different name will cause the operator to silently fall back to defaults.

Template ALL AdminConfig fields from values.yaml -> ConfigMap data, matching the keys the Go code reads via LoadAdminConfig:
- maxServersGlobal, maxServersPerUser (from adminConfig.limits)
- quotaCPURequests, quotaCPULimits, quotaMemoryRequests, quotaMemoryLimits, quotaPods, quotaPVCs, quotaStorage (from adminConfig.quota)
- defaultCPU, defaultMemory, defaultRequestCPU, defaultRequestMemory, maxCPU, maxMemory, minCPU, minMemory (from adminConfig.containerDefaults)
- baseDomain, gatewayName, gatewayNamespace (use Release.Namespace if value empty), gatewayControllerNamespace (from adminConfig.networking)
- jwtExpirationHours, inviteExpirationHours, registrationEnabled, panelURL (from adminConfig.auth)
- smtpHost, smtpPort, smtpUsername, smtpFrom (from adminConfig.smtp, only if smtpHost is set)
- modStorageClass, modStorageSize (from adminConfig.storage)
- backupS3Endpoint, backupS3Bucket, backupS3Region, backupS3UseSSL, backupRetentionCount (from adminConfig.backup, only if backup.enabled)

All values must be quoted strings (Go code parses them from ConfigMap string data).
