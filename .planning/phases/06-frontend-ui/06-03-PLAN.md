       - `listServers(): Promise<ListResponse<GameServerResponse>>` -- GET /gameservers
       - `getServer(name: string): Promise<GameServerResponse>` -- GET /gameservers/{name}
       - `createServer(data: CreateGameServerRequest): Promise<GameServerResponse>` -- POST /gameservers
       - `updateServer(name: string, data: UpdateGameServerRequest): Promise<GameServerResponse>` -- PUT /gameservers/{name}
       - `deleteServer(name: string): Promise<void>` -- DELETE /gameservers/{name}
       - `startServer(name: string): Promise<GameServerResponse>` -- POST /gameservers/{name}/start
       - `stopServer(name: string): Promise<GameServerResponse>` -- POST /gameservers/{name}/stop
       - `restartServer(name: string): Promise<GameServerResponse>` -- POST /gameservers/{name}/restart

    3. **Create `web/src/hooks/use-servers.ts`** -- TanStack Query hooks:
       - `useServers()` -- useQuery with queryKey ['servers'], refetchInterval: 5000 (poll every 5s for status updates)
       - `useServer(name: string)` -- useQuery with queryKey ['servers', name], refetchInterval: 2000 (faster polling for detail page), enabled: !!name
       - `useCreateServer()` -- useMutation, onSuccess invalidates ['servers']
       - `useDeleteServer()` -- useMutation, onSuccess invalidates ['servers']
       - `useStartServer()` -- useMutation, onSuccess invalidates ['servers'] and ['servers', name]
       - `useStopServer()` -- useMutation, onSuccess invalidates ['servers'] and ['servers', name]
       - `useRestartServer()` -- useMutation, onSuccess invalidates ['servers'] and ['servers', name]
       All mutations should show toast on error via sonner's `toast.error()`.

    4. **Create `web/src/components/servers/server-status-badge.tsx`**:
       - Takes `state` prop (GameServerResponse['state'])
       - Maps state to color: Creating=blue, Starting=yellow, Ready=green, Allocated=purple, Shutdown=gray, Error=red
       - Uses shadcn Badge component with appropriate variant/color

    5. **Create `web/src/components/forms/game-config-form.tsx`**:
       - Import `withTheme` from `@rjsf/core`, `Theme as ShadcnTheme` from `@rjsf/shadcn`, `validator` from `@rjsf/validator-ajv8`
       - Create `Form` via `withTheme(ShadcnTheme)` at module level
       - Props: `parameterSchema: RJSFSchema`, `defaultParameters: Record<string, string>`, `onSubmit: (parameters: Record<string, string>) => void`, `isLoading?: boolean`
       - Render `<Form schema={parameterSchema} formData={defaultParameters} validator={validator} onSubmit={({ formData }) => onSubmit(formData)} />`
       - NOTE: Do NOT configure Ajv for draft-2020-12. The default draft-07 validator works because the schemas only use draft-07 features (enum, const, pattern, maxLength, default, type, required). See 06-RESEARCH.md Pitfall 3.
       - Add a submit button below the form (shadcn Button with loading state)
       - If parameterSchema is empty/undefined, show a message "No configurable parameters" and just a submit button
