  <action>
    1. **Add shadcn/ui components needed for auth and layout:**
       ```bash
       cd web && npx shadcn@latest add button input label card toast sidebar
       ```
       This generates components into `src/components/ui/`.

    2. **Create `web/src/api/auth.ts`** -- API functions for auth endpoints:
       - `login(data: LoginRequest): Promise<LoginResponse>` -- POST /auth/login
       - `register(data: RegisterRequest): Promise<RegisterResponse>` -- POST /auth/register
       - `refreshToken(): Promise<LoginResponse>` -- POST /auth/refresh
       - All use `apiFetch` from client.ts

    3. **Create `web/src/hooks/use-auth.ts`** -- TanStack Query mutations for auth:
       - `useLogin()` -- useMutation calling login API, on success calls `useAuthStore.getState().setToken()`
       - `useRegister()` -- useMutation calling register API, on success calls setToken
       - `useLogout()` -- returns function that calls `useAuthStore.getState().logout()` and navigates to /login

    4. **Create `web/src/components/auth/protected-route.tsx`**:
       - `ProtectedRoute` -- checks `useAuthStore(s => s.token)`, redirects to `/login` if null, renders `<Outlet />` otherwise
       - `AdminRoute` -- checks `useAuthStore(s => s.user?.role)`, redirects to `/` if not 'admin', renders `<Outlet />` otherwise

    5. **Create `web/src/pages/login.tsx`**:
       - Card-based form with username and password inputs
       - Submit calls `useLogin()` mutation
       - On success, navigate to `/` (dashboard)
       - Show error toast on failure via sonner
       - Link to register page ("Have an invite? Register here")
       - Kterodactyl branding/title

    6. **Create `web/src/pages/register.tsx`**:
       - Card-based form with username, email, password, invite token inputs
       - Submit calls `useRegister()` mutation
       - On success, navigate to `/`
       - Show error toast on failure
       - Link back to login page

    7. **Create `web/src/components/layout/header.tsx`**:
       - Displays current user's username and role (from auth store)
       - Logout button calling useLogout
       - "Kterodactyl" title/branding

    8. **Create `web/src/components/layout/sidebar.tsx`**:
       - Navigation links: Dashboard (/), Games (/games), My Servers (/servers)
       - Admin section (visible only if user.role === 'admin'): Users (/admin/users), Invites (/admin/invites)
       - Use lucide-react icons (Gamepad2, Server, Users, Mail, LayoutDashboard)
       - Highlight active route using react-router's useLocation

    9. **Create `web/src/components/layout/app-shell.tsx`**:
       - Layout wrapper: sidebar on left, header on top, main content area
       - Renders `<Outlet />` in the main content area
       - Responsive: sidebar collapsible on mobile

    10. **Create `web/src/pages/dashboard.tsx`**:
        - Simple welcome page showing user info
        - Quick stats placeholder (server count -- will wire to real data in Plan 03)
        - "Create Server" call-to-action button linking to /servers/create

    11. **Update `web/src/App.tsx`** with full route tree:
        ```
        /login -- LoginPage (public)
        /register -- RegisterPage (public)
        / -- ProtectedRoute > AppShell
          / -- Dashboard
          /games -- GamesPage (Plan 02 Task 2)
          /servers -- (placeholder for Plan 03)
          /servers/create -- (placeholder for Plan 03)
          /servers/:name -- (placeholder for Plan 03)
          /admin -- AdminRoute
            /admin/users -- (placeholder for Plan 04)
            /admin/invites -- (placeholder for Plan 04)
        ```
        Use react-router v7 `createBrowserRouter` + `RouterProvider` pattern, or `<BrowserRouter>` + `<Routes>`.

    12. **Update `web/src/main.tsx`**:
        - Add `<Toaster />` from sonner for toast notifications
        - Keep `<QueryClientProvider>` wrapping the app

       - `listGames(): Promise<ListResponse<GameResponse>>` -- GET /games
       - `getGame(gameType: string): Promise<GameResponse>` -- GET /games/{gameType}

    3. **Create `web/src/hooks/use-games.ts`**:
       - `useGames()` -- `useQuery({ queryKey: ['games'], queryFn: listGames })` with staleTime of 5 minutes (game list changes rarely)
       - `useGame(gameType: string)` -- `useQuery({ queryKey: ['games', gameType], queryFn: () => getGame(gameType), enabled: !!gameType })`

    4. **Create `web/src/components/games/game-card.tsx`**:
       - Card displaying: game displayName, game type (badge), port info, number of configurable parameters

       - Use shadcn Card + Badge components

    5. **Create `web/src/components/games/game-list.tsx`**:

       - Maps game data to GameCard components

    6. **Create `web/src/pages/games.tsx`**:
