  <action>
    Add three new lifecycle endpoints that the frontend needs for server management:

    1. **In `internal/api/handlers_gameserver.go`**, add three handlers:

       **`handleStopGameServer`** (POST /api/v1/gameservers/{name}/stop):
       - Get namespace from context, get GameServer by name
       - If already in Shutdown state, return 409 Conflict "server is already stopped"
       - If in terminal/error state, return 409 Conflict with appropriate message
       - Set `Status.State` to `GameServerStateShutdown` via status sub-resource update
       - The operator will handle Pod cleanup on next reconciliation
       - Return 200 with updated GameServerResponse

       **`handleRestartGameServer`** (POST /api/v1/gameservers/{name}/restart):
       - Get namespace from context, get GameServer by name
       - If in Shutdown or Error state, set state back to Creating (the operator reconciler handles Creating by creating a Pod)
       - If in Ready/Allocated state, delete the underlying Pod (operator recreates it) -- look up Pod via owner label, delete it. The operator's reconciler handles Starting->Creating transition when Pod disappears.
       - Return 200 with updated GameServerResponse

       **`handleStartGameServer`** (POST /api/v1/gameservers/{name}/start):
       - Get namespace from context, get GameServer by name
       - If server is already in Creating/Starting/Ready/Allocated, return 409 "server is already running"
       - If in Shutdown or Error state, set state back to Creating
       - Return 200 with updated GameServerResponse

       For status updates, use the pattern: `s.client.Status().Update(ctx, gs)` after modifying `gs.Status.State`.

       IMPORTANT: Import `gamev1alpha1` for state constants. Use existing `gameServerToResponse()` for response mapping. Follow the existing handler patterns (namespace from context, 404 on not found, error handling).

    2. **In `internal/api/routes.go`**, register the three new routes inside the `/{name}` route group:
       ```go
       r.Post("/start", s.handleStartGameServer)
       r.Post("/stop", s.handleStopGameServer)
       r.Post("/restart", s.handleRestartGameServer)
       ```

    3. **In `internal/api/handlers_gameserver_test.go`**, add tests:
       - `TestHandleStopGameServer` -- stop a Ready server, verify 200 and state=Shutdown; stop an already-stopped server, verify 409
       - `TestHandleStartGameServer` -- start a Shutdown server, verify 200 and state=Creating; start a Ready server, verify 409
       - `TestHandleRestartGameServer` -- restart a Ready server, verify 200
       - Follow existing test patterns: use `testServer`, `generateToken`, `doRequest`, and `createTestGameServer` helpers from `helpers_test.go`
  </action>
