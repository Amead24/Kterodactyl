  <action>
    1. **Create `internal/api/spa.go`** with embedded frontend:
       ```go
       package api

       import (
           "embed"
           "io"
           "io/fs"
           "net/http"
           "strings"
       )

       //go:embed all:frontend
       var frontendFS embed.FS

       func serveSPA() http.Handler {
           distFS, _ := fs.Sub(frontendFS, "frontend")
           fileServer := http.FileServer(http.FS(distFS))

           return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
               path := strings.TrimPrefix(r.URL.Path, "/")

               // Try to open the file in the embedded FS
               f, err := distFS.Open(path)
               if err != nil {
                   // File not found: serve index.html for client-side routing
                   indexFile, _ := distFS.Open("index.html")
                   defer indexFile.Close()
                   stat, _ := indexFile.Stat()
                   http.ServeContent(w, r, "index.html", stat.ModTime(), indexFile.(io.ReadSeeker))
                   return
               }
               f.Close()
               fileServer.ServeHTTP(w, r)
           })
       }
       ```

       NOTE on embed path: The `//go:embed` directive path is relative to the Go source file. Since the file is in `internal/api/`, the embedded directory will be `internal/api/frontend/`. The build pipeline must copy `web/dist/*` to `internal/api/frontend/` before `go build`. This avoids complex relative path issues.

       ALTERNATIVE: If you prefer, place the embed directive at the project root in a new file and pass the handler into the Server. Choose whichever is cleaner -- the critical requirement is that `go:embed` finds the built assets at compile time.

    2. **Update `internal/api/routes.go`** to register the SPA catch-all:

       - Add: `r.NotFound(serveSPA().ServeHTTP)` -- this serves the SPA for any route not matched by API handlers
       - This ensures API routes always take priority over the SPA fallback
       - IMPORTANT: Do NOT remove or modify existing API routes or middleware

       - Update `build` target to depend on `build-frontend`:

       - Add `dev-frontend` target for development:


    4. **Update `Dockerfile`** to include Node.js build stage:
       ```dockerfile
       # Build frontend static assets
       FROM node:22-alpine AS frontend
       WORKDIR /web
       COPY web/package.json web/package-lock.json ./
       RUN npm ci
       COPY web/ .
       RUN npm run build

       # Build the manager binary
       FROM golang:1.25 AS builder
       ARG TARGETOS
       ARG TARGETARCH
       WORKDIR /workspace
       COPY go.mod go.sum ./
       RUN go mod download
       COPY . .
       # Copy frontend build output to embed location
       COPY --from=frontend /web/dist ./internal/api/frontend
       RUN CGO_ENABLED=0 GOOS=${TARGETOS:-linux} GOARCH=${TARGETARCH} go build -a -o manager cmd/main.go

       # Production image
       FROM gcr.io/distroless/static:nonroot
       WORKDIR /
       COPY --from=builder /workspace/manager .
       COPY --from=builder /workspace/games /games
       USER 65532:65532
       ENTRYPOINT ["/manager"]
       ```
