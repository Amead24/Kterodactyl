**Create `internal/api/handlers_mods.go`:**

This new file contains three handlers:

1. **`handleUploadMod`** (POST /gameservers/{name}/mods):
   - Get server name from `chi.URLParam(r, "name")` and namespace from `namespaceFromContext(r)`
   - Fetch the GameServer CR; verify it exists (404 if not)
   - Check state is Ready or Allocated; return 409 Conflict with "server must be running to upload mods" otherwise
   - Read `AnnotationModPath` from the GameServer; return 400 "game does not support mods" if empty
   - Apply `http.MaxBytesReader(w, r.Body, 100<<20)` for 100MB upload limit
   - Call `r.ParseMultipartForm(32 << 20)`; on error, return 400 "file too large or invalid form data"
   - Get file via `r.FormFile("file")`; defer close
   - Sanitize filename: `filepath.Base(header.Filename)`, reject if "." or ".." or contains "/" or "\\"
   - Find the pod name (same as GameServer name) in the GameServer's namespace
   - Use `io.Pipe()` to connect tar writer to exec stdin:
     - Goroutine: create `tar.NewWriter(writer)`, write header with `Name: filename, Size: header.Size, Mode: 0644, ModTime: time.Now()`, then `io.Copy(tw, file)`, close tw, close writer
     - Main: build SPDY exec request to pod, command `["tar", "-xf", "-", "-C", modPath]`, stdin=reader, capture stdout/stderr
     - Use `remotecommand.NewSPDYExecutor(s.restConfig, "POST", req.URL())` same pattern as console handler
   - On exec error, return 500 with stderr output
   - After successful upload, trigger server restart:
     - Re-fetch the GameServer (re-fetch before status update pattern)
     - Set `gs.Status.State = gamev1alpha1.GameServerStateCreating` via `Status().Update()`
     - This is safe because the state machine has Ready->Creating and Allocated->Creating transitions (verify this or add them)
   - Return 200 with JSON: `{"name": filename, "size": header.Size}`

   Response type:
   ```go
   type ModFileResponse struct {
       Name string `json:"name"`
       Size int64  `json:"size"`
   }
   ```

2. **`handleListMods`** (GET /gameservers/{name}/mods):
   - Same auth/namespace/state checks as upload (state must be Ready or Allocated)
   - Read modPath from annotation
   - Execute `["ls", "-la", modPath]` in the pod via remotecommand exec
   - Parse the ls output: skip first line (total), split remaining lines, extract filename (last field) and size (5th field)
   - Return JSON: `{"data": [{"name": "...", "size": 123}, ...], "count": N}`
   - If the directory is empty or ls fails (e.g., no files), return empty list

3. **`handleDeleteMod`** (DELETE /gameservers/{name}/mods/{filename}):
   - Get filename from `chi.URLParam(r, "filename")`
   - Sanitize filename same as upload (reject "..", "/", "\\")
   - Same auth/namespace/state checks
   - Execute `["rm", "-f", modPath + "/" + filename]` in the pod via remotecommand exec
   - Return 204 No Content on success

**Shared helper** (in handlers_mods.go):

func (s *Server) execInPod(ctx context.Context, namespace, podName string, command []string, stdin io.Reader) (string, string, error)
```
Returns stdout, stderr, error. The upload handler passes the tar pipe as stdin; list and delete pass nil for stdin.

// Mod management
r.Route("/mods", func(r chi.Router) {
    r.Post("/", s.handleUploadMod)
    r.Get("/", s.handleListMods)
    r.Delete("/{filename}", s.handleDeleteMod)
})
```

This places mod routes at:
- POST /api/v1/gameservers/{name}/mods -- upload
- GET /api/v1/gameservers/{name}/mods -- list
