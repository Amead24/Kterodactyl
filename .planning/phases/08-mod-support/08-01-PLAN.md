<task type="auto">
  <name>Task 2: Add PVC creation and volume mounting to controller</name>
  <files>internal/controller/gameserver_controller.go</files>
  <action>
In `internal/controller/gameserver_controller.go`:

1. **Add RBAC marker** for PersistentVolumeClaims. Add this line alongside the existing RBAC markers (near line 268):
   ```
   // +kubebuilder:rbac:groups="",resources=persistentvolumeclaims,verbs=get;list;watch;create;update;patch;delete
   ```

2. **Add AdminConfig fields** for mod storage. In the `AdminConfig` struct, add after the SMTP section:
   ```go
   // Mod storage
   ModStorageSize  resource.Quantity // PVC size for mod storage per server (default: 1Gi)
   ModStorageClass string            // StorageClass for mod PVCs (empty = cluster default)
   ```
   In `DefaultAdminConfig()`, add:
   ```go
   ModStorageSize:  resource.MustParse("1Gi"),
   ModStorageClass: "",
   ```
   In `LoadAdminConfig`, add parsing for these fields:
   - `modStorageSize`: parse as `resource.Quantity` via `resource.ParseQuantity(v)`, overwrite `cfg.ModStorageSize`
   - `modStorageClass`: string field, set directly to `cfg.ModStorageClass`

3. **Add `reconcileModPVC` method** on `GameServerReconciler`. This creates or updates a PVC for mod storage:
   - PVC name: `gs.Name + "-mods"`
   - PVC namespace: `gs.Namespace`
   - Use `controllerutil.CreateOrUpdate` pattern (same as reconcilePod)
   - Set `ctrl.SetControllerReference(gs, pvc, r.Scheme)` for garbage collection
   - Spec: `AccessModes: []corev1.PersistentVolumeAccessMode{corev1.ReadWriteOnce}`
   - Spec: `Resources.Requests[corev1.ResourceStorage] = modStorageSize` (from AdminConfig)
   - If `modStorageClass` is non-empty, set `pvc.Spec.StorageClassName = &modStorageClass`
   - **IMPORTANT**: Only set spec fields on creation (when pvc.CreationTimestamp.IsZero()), not on update. PVC specs are immutable after creation.
   - Signature: `func (r *GameServerReconciler) reconcileModPVC(ctx context.Context, gs *gamev1alpha1.GameServer, cfg *AdminConfig) error`
   - Log the result (Created/Unchanged) and record an event.

4. **Modify `reconcileCreating`** to call `reconcileModPVC` BEFORE `reconcilePod`. The PVC must exist before the pod references it. Insert the call after `ensureUserNamespace` and before `reconcilePod`:
   ```go
   // Create mod storage PVC if the game supports mods
   // Load the game manifest to check for modPath
   // Note: manifestLoader is not available in the controller. Instead, accept that
   // the API handler already validated the gameType. The modPath comes from the manifest
   // at server creation time. We need a way to know the modPath in the controller.
   ```

   Actually, the controller does NOT have access to the manifest loader. The modPath needs to be stored on the GameServer spec or as an annotation. The cleanest approach: **store modPath as an annotation on the GameServer** at creation time (set by the API handler in handlers_gameserver.go). The controller reads the annotation.

   Define a new annotation constant. In `internal/util/labels.go` (or add at top of controller file):
   ```go
   const AnnotationModPath = "kterodactyl.io/mod-path"
   ```

   In `reconcileCreating`, after `ensureUserNamespace`:
   ```go
   modPath := gs.Annotations[AnnotationModPath] // empty if game doesn't support mods
   if modPath != "" {
       if err := r.reconcileModPVC(ctx, gs, adminCfg); err != nil {
           return ctrl.Result{}, fmt.Errorf("failed to reconcile mod PVC: %w", err)
       }
   }
   ```

5. **Modify `reconcilePod`** to add volume and volume mount when modPath annotation is present:
   After building the pod spec (the `pod.Spec = corev1.PodSpec{...}` block), add:
   ```go
   // Add mod storage volume mount if game supports mods
   if modPath := gs.Annotations[AnnotationModPath]; modPath != "" {
       pod.Spec.Volumes = append(pod.Spec.Volumes, corev1.Volume{
           Name: "mods",
           VolumeSource: corev1.VolumeSource{
               PersistentVolumeClaim: &corev1.PersistentVolumeClaimVolumeSource{
                   ClaimName: gs.Name + "-mods",
               },
           },
       })
       pod.Spec.Containers[0].VolumeMounts = append(pod.Spec.Containers[0].VolumeMounts, corev1.VolumeMount{
           Name:      "mods",
           MountPath: modPath,
       })
   }
   ```
   This must be INSIDE the CreateOrUpdate mutate function, after `pod.Spec = corev1.PodSpec{...}`.

6. **Add the AnnotationModPath constant** in `internal/util/labels.go` alongside the existing annotation constants. If there's no labels.go, add it at the top of the controller file in a const block.

Run `go build ./...` to verify compilation.
