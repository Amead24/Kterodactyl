Create `internal/api/handlers_mods_test.go` in `package api` following the established per-subtest fresh server pattern from `handlers_gameserver_test.go`.

**Test functions to create:**

1. **TestHandleListMods** with subtests:
   - `"server not found returns 404"` -- GET /api/v1/gameservers/nonexistent/mods with auth
   - `"server not running returns 409"` -- create GameServer in Shutdown state, GET /mods
   - `"no mod path returns 400"` -- create Ready GameServer without mod path annotation, GET /mods
   - `"unauthenticated returns 401"` -- GET /mods without auth header

2. **TestHandleUploadMod** with subtests:
   - `"server not found returns 404"` -- POST /api/v1/gameservers/nonexistent/mods with auth
   - `"server not running returns 409"` -- create Shutdown GameServer, POST /mods
   - `"no mod path returns 400"` -- create Ready GameServer without annotation, POST /mods
   - Note: Cannot test the full upload happy path (requires execInPod). Test only validation-layer paths per user decision. Use mock/pre-built request bodies (not real multipart form data) per user decision.

3. **TestHandleDeleteMod** with subtests:
   - `"server not found returns 404"` -- DELETE /api/v1/gameservers/nonexistent/mods/test.jar with auth
   - `"server not running returns 409"` -- create Shutdown GameServer, DELETE /mods/test.jar
   - `"no mod path returns 400"` -- create Ready GameServer without annotation, DELETE /mods/test.jar

4. **TestParseLsOutput** (pure function, no server needed):
   - `"parses standard ls -la output"` -- input with "total" line, files, directories; verify ModFileResponse names and sizes
   - `"skips dot directories"` -- input with . and ..; verify they are excluded
   - `"empty output"` -- empty string returns nil/empty slice
   - `"single file"` -- single file line returns one entry

**Implementation rules (per locked decisions):**
- Error case assertions use HTTP status codes ONLY (e.g., `if rec.Code != http.StatusNotFound`)
- Do NOT assert on error message text
- For upload endpoint: use `httptest.NewRequest` with nil or bytes.Reader body -- do NOT construct real multipart form data. The handler will hit the validation paths before reaching ParseMultipartForm.
- Each `t.Run` gets its own `newTestServer(t)` -- zero shared state
- Use `createTestGameServerWithState` for non-running server tests
- Use `createTestGameServerWithAnnotations` with `util.AnnotationModPath: "/mods"` for mod-path tests when testing paths beyond the annotation check (not needed for current scope since we test absence of annotation)

**Route paths (from routes.go):**
- POST `/api/v1/gameservers/{name}/mods` -> handleUploadMod
- GET `/api/v1/gameservers/{name}/mods` -> handleListMods
- DELETE `/api/v1/gameservers/{name}/mods/{filename}` -> handleDeleteMod

Note: chi URL parameters are set via the router, so use the full path in `httptest.NewRequest`.
