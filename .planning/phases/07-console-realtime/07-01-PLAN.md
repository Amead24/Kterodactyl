<task type="auto">
  <name>Task 2: Implement WebSocket console handler with log streaming and command exec</name>
  <files>internal/api/handlers_console.go</files>
  <action>
  Create `internal/api/handlers_console.go` implementing `handleConsole(w, r)` as a WebSocket endpoint:

  1. **WebSocket upgrader:** Configure gorilla/websocket Upgrader with 1024 read/write buffers. CheckOrigin returns true (same-origin SPA via go:embed).

  2. **Authentication via query param:** Extract JWT token from `r.URL.Query().Get("token")`. Validate using `s.jwtService.ValidateToken(token)`. Return 401 if invalid. Extract namespace from claims.

  3. **Server ownership check:** Fetch the GameServer CR using `s.client.Get()` with the name from `chi.URLParam(r, "name")` and namespace from claims. Return 404 if not found.

  4. **State check:** Only allow console when state is Ready or Allocated. Return 409 Conflict with "server not running" otherwise.

  5. **WebSocket upgrade:** Call `upgrader.Upgrade(w, r, nil)`. Set up ping/pong keepalive:
     - Set PongHandler to update read deadline
     - Start a goroutine that sends Ping every 30 seconds (for Cloudflare Tunnel compatibility)

  6. **Write channel pattern:** Create a `writeCh := make(chan []byte, 256)` and a dedicated writer goroutine that reads from the channel and calls `conn.WriteMessage(websocket.TextMessage, msg)`. This prevents concurrent write panics (Pitfall 3 from research).

  7. **Log streaming goroutine:** In a separate goroutine, call:
     ```go
     req := s.clientset.CoreV1().Pods(namespace).GetLogs(serverName, &corev1.PodLogOptions{
         Container: "gameserver",
         Follow:    true,
         TailLines: int64Ptr(100),
     })
     stream, err := req.Stream(ctx)
     ```
     Read from stream in a loop (4096 byte buffer) and send to `writeCh`. On EOF, send a JSON `{"type":"stream_ended"}` message. Use the shared context for cancellation.

  8. **Command reading loop:** In the main goroutine (after starting log streaming), read WebSocket messages. Parse JSON messages with `{"type":"command","data":"..."}`. For command messages, execute via remotecommand:
     ```go
     req := s.clientset.CoreV1().RESTClient().Post().
         Resource("pods").Name(serverName).Namespace(namespace).
         SubResource("exec").
         VersionedParams(&corev1.PodExecOptions{
             Container: "gameserver",
             Command:   []string{"/bin/sh", "-c", fmt.Sprintf("echo '%s' > /proc/1/fd/0", command)},
             Stdin:     false, Stdout: true, Stderr: true,
         }, scheme.ParameterCodec)
     exec, _ := remotecommand.NewSPDYExecutor(s.restConfig, "POST", req.URL())
     exec.StreamWithContext(ctx, remotecommand.StreamOptions{Stdout: &stdout, Stderr: &stderr})
     ```
     Send any exec output/errors back through `writeCh`.

  9. **Cleanup:** Use `context.WithCancel(r.Context())`. Cancel the context when:
     - WebSocket read returns error (client disconnect)
     - Close message received
     - Server shutdown (parent context cancelled)
     Close `writeCh` after cancellation to stop the writer goroutine.

  10. **Send initial connected message:** After upgrade, send `{"type":"connected","data":"streaming logs..."}` via writeCh.

  Helper: `int64Ptr(i int64) *int64` returns a pointer to the int64.

  Use `"runtime/scheme"` from the existing k8s client-go for ParameterCodec -- specifically `k8s.io/client-go/kubernetes/scheme`.
  </action>

           MemoryMiB      int64  `json:"memoryMiB"`      // MiB

     - Extract CPU from `container.Usage.Cpu().MilliValue()` and memory from `container.Usage.Memory().Value() / (1024 * 1024)`.
     - Also fetch the GameServer spec to get resource limits (cpuLimit, memoryLimit) for display context.
     - Handle errors gracefully: if metrics API is unavailable (e.g., no metrics-server), return 503 with "metrics unavailable" message rather than 500.
     - Use the first container in the metrics response (the "gameserver" container).

  2. Restructure `internal/api/routes.go` to separate WebSocket routes from timeout middleware:

     **CRITICAL (Pitfall 1 from research):** The existing `middleware.Timeout(30 * time.Second)` on line 39 is applied globally and WILL kill WebSocket connections after 30 seconds.

     Restructure the router:
     - Remove `middleware.Timeout(30 * time.Second)` from the global middleware stack (line 39).
     - Add it ONLY to the REST API route group instead.

     New routes() structure:
     ```go
     func (s *Server) routes() chi.Router {
         r := chi.NewRouter()

         // Global middleware (no timeout -- WebSocket routes are long-lived)
         r.Use(middleware.RequestID)
         r.Use(middleware.RealIP)
         r.Use(middleware.Logger)
         r.Use(middleware.Recoverer)
         r.Use(cors.Handler(cors.Options{...}))  // keep existing CORS config
         r.Use(httprate.LimitByIP(100, time.Minute))

         // Health routes
         r.Get("/healthz", handleHealthz)
         r.Get("/readyz", handleReadyz)

         // Public auth routes (keep existing rate limits)
         r.With(httprate.LimitByIP(5, time.Minute)).Post("/api/v1/auth/login", s.handleLogin)
         r.With(httprate.LimitByIP(3, time.Minute)).Post("/api/v1/auth/register", s.handleRegister)

         // WebSocket routes (NO timeout, auth handled in handler via query param)
         r.Get("/api/v1/gameservers/{name}/console", s.handleConsole)

         // REST API routes WITH timeout
         r.Route("/api/v1", func(r chi.Router) {
             r.Use(middleware.Timeout(30 * time.Second))
             r.Use(s.authMiddleware.Authenticate)

             // Auth management
             r.Post("/auth/refresh", s.handleRefresh)

             // Games
             r.Get("/games", s.handleListGames)
             r.Get("/games/{gameType}", s.handleGetGame)

             // GameServers
             r.Route("/gameservers", func(r chi.Router) {
                 r.Get("/", s.handleListGameServers)
                 r.With(httprate.LimitByIP(10, time.Minute)).Post("/", s.handleCreateGameServer)
                 r.Route("/{name}", func(r chi.Router) {
                     r.Get("/", s.handleGetGameServer)
                     r.Put("/", s.handleUpdateGameServer)
                     r.Delete("/", s.handleDeleteGameServer)
                     r.Post("/start", s.handleStartGameServer)
                     r.Post("/stop", s.handleStopGameServer)
                     r.Post("/restart", s.handleRestartGameServer)
                     r.Get("/metrics", s.handleGetMetrics)
                 })
             })

             // Admin routes
             r.Route("/admin", func(r chi.Router) {
                 r.Use(auth.RequireAdmin)
                 r.Post("/invites", s.handleCreateInvite)
                 r.Get("/users", s.handleListUsers)
                 r.Delete("/users/{username}", s.handleDeleteUser)
             })
         })

         r.NotFound(serveSPA().ServeHTTP)
         return r
     }
     ```

     Key changes:
     - `middleware.Timeout` moved from global to REST route group only
     - WebSocket console route registered at top level (before /api/v1 group) with no timeout
     - Metrics endpoint added inside existing authenticated gameserver routes
     - WebSocket route does NOT use authMiddleware.Authenticate (auth is in the handler via query param)
