  <files>web/package.json, web/src/hooks/use-console.ts, web/src/components/console/terminal.tsx, web/src/components/console/console-panel.tsx</files>
  <action>
  1. Install frontend dependencies:
     ```bash
     cd web && npm install @xterm/xterm @xterm/addon-fit
     ```

  2. Create `web/src/hooks/use-console.ts` -- custom WebSocket hook (NOT react-use-websocket due to React 19 peer dep conflict):
     - Export `ConnectionStatus` type: `'connecting' | 'connected' | 'disconnected' | 'error'`
     - Accept options: `{ serverName: string, onMessage: (data: string) => void, enabled?: boolean }`
     - On connect:
       - Get token from `useAuthStore.getState().token`
       - Build URL: `${protocol}//${window.location.host}/api/v1/gameservers/${serverName}/console?token=${token}` where protocol is `wss:` for HTTPS, `ws:` for HTTP
       - Create `new WebSocket(url)`
     - Handle onopen: set status to 'connected', reset reconnect count
     - Handle onmessage: call `onMessage(event.data)`
     - Handle onclose: if not intentional (code !== 1000) and reconnect count < 5, reconnect with exponential backoff (1s, 2s, 4s, 8s, 16s, capped at 30s)
     - Handle onerror: set status to 'error'
     - Return `{ status, sendCommand, disconnect }` where:
       - `sendCommand(cmd: string)` sends `JSON.stringify({ type: 'command', data: cmd })` if WS is OPEN
       - `disconnect()` prevents reconnect and closes with code 1000
     - useEffect: connect when enabled, cleanup on unmount (close WS, clear timeout)
     - Use useCallback for connect/sendCommand/disconnect to prevent re-renders
     - Use useRef for ws, reconnect timeout, and reconnect count

  3. Create `web/src/components/console/terminal.tsx` -- xterm.js terminal component:
     - Import `Terminal` from `@xterm/xterm`, `FitAddon` from `@xterm/addon-fit`, and the CSS `@xterm/xterm/css/xterm.css`
     - Accept props: `{ onData?: (data: string) => void }` (for typing input)
     - Expose a `write(data: string)` method via `useImperativeHandle` with a `React.forwardRef` pattern so the parent can write data to the terminal
     - useEffect on mount:
       - Create `new Terminal({ cursorBlink: true, fontSize: 14, fontFamily: 'JetBrains Mono, Fira Code, monospace', theme: { background: '#1a1b26', foreground: '#a9b1d6' }, scrollback: 5000, convertEol: true })`
       - Load FitAddon, open terminal on container div, call fit()
       - Wire `terminal.onData(onData)` if provided
       - Use ResizeObserver on container to call `fitAddon.fit()` on resize
       - Cleanup: disconnect observer, dispose terminal
     - Render: `<div ref={containerRef} className="h-full w-full" />`
     - Export the ref handle type: `TerminalHandle { write: (data: string) => void }`

  4. Create `web/src/components/console/console-panel.tsx` -- combines terminal with connection status:
     - Accept props: `{ serverName: string, enabled: boolean }` (enabled = server is active)
     - Use a ref to `TerminalHandle`
     - Use `useConsole({ serverName, onMessage: (data) => terminalRef.current?.write(data), enabled })`
     - Show connection status indicator: green dot for connected, yellow for connecting, red for disconnected/error
     - Show a command input field at the bottom (HTML input, not xterm onData) for cleaner UX:
       - On Enter key: call `sendCommand(inputValue)`, clear input
       - Disabled when not connected
     - Layout: flex column, terminal takes remaining space, input bar at bottom
     - When not enabled, show a centered message: "Console available when server is running"
  </action>
