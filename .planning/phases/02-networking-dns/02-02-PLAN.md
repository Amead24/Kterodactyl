<objective>
Implement the DNS controller that watches GameServer resources and creates per-server Service + HTTPRoute resources for DNS routing.

Purpose: This is the core of Phase 2. When a GameServer reaches Ready or Allocated state, the DNS controller creates a ClusterIP Service (for HTTPRoute backend) and an HTTPRoute with the hostname `game.username.baseDomain`. It also updates the GameServer status with connection info (DNS name and ports). ExternalDNS (deployed externally) watches the HTTPRoute hostnames and provisions DNS records automatically.

Output: Working DNS controller that creates networking resources per GameServer and populates connection info in status.

Create `internal/controller/dns_controller.go` implementing a second reconciler in the same operator binary. Follow the patterns established in Phase 1 (gameserver_controller.go) and the architecture from 02-RESEARCH.md.

**DNSReconciler struct:**
```go
type DNSReconciler struct {
    client.Client
    Scheme            *runtime.Scheme
    Recorder          record.EventRecorder
    OperatorNamespace string
}
```
Note: Do NOT store BaseDomain/GatewayName as struct fields. Load them from AdminConfig on each reconciliation (same pattern as gameserver_controller.go) so they can be changed without operator restart.

**Reconcile method flow:**
1. Fetch the GameServer CR. Return `client.IgnoreNotFound` on 404.
2. Load AdminConfig via `LoadAdminConfig(ctx, r.Client, r.operatorNs())`. If `adminCfg.BaseDomain == ""`, skip networking (DNS not configured). Log a warning once and return.
3. If GameServer is being deleted (DeletionTimestamp not zero), return -- owner references handle cleanup automatically.
4. If GameServer state is NOT Ready and NOT Allocated:
   - Delete the Service and HTTPRoute if they exist (cleanup when server stops). Use `client.IgnoreNotFound` on delete errors.
   - Clear `status.Address` and `status.Ports` on the GameServer if they are set (re-fetch before status update to avoid conflicts). Return.
5. Extract owner from `gs.Labels[util.LabelOwner]`. If empty, log error and return (do not create networking for ownerless servers).
6. Build DNS name: `util.GameServerDNSName(gs.Spec.GameType, owner, adminCfg.BaseDomain)`
7. Call `r.ensureService(ctx, gs)` -- creates/updates ClusterIP Service
8. Call `r.ensureHTTPRoute(ctx, gs, dnsName, adminCfg)` -- creates/updates HTTPRoute
9. Call `r.updateConnectionInfo(ctx, gs, dnsName)` -- updates GameServer status with DNS name and ports

**ensureService method:**
- Use `controllerutil.CreateOrUpdate` pattern (same as Phase 1)
- Service name: same as GameServer name (`gs.Name`)
- Service namespace: same as GameServer namespace (`gs.Namespace`)
- Service type: `corev1.ServiceTypeClusterIP`
- Set owner reference via `ctrl.SetControllerReference(gs, svc, r.Scheme)`
- Labels: `util.LabelManagedBy: util.ManagedByValue`, `util.LabelHTTPRouteOwner: gs.Name` (from networking.go)
- Selector: `util.LabelOwner: owner`, `util.LabelGame: gs.Spec.GameType`, `util.LabelName: util.AppNameValue`
- Ports: map from `gs.Spec.Ports` to ServicePorts (Name, Port=ContainerPort, TargetPort=ContainerPort, Protocol)
- Return error

**ensureHTTPRoute method:**
- Use `controllerutil.CreateOrUpdate`
- HTTPRoute name: same as GameServer name
- HTTPRoute namespace: same as GameServer namespace
- Set owner reference via `ctrl.SetControllerReference(gs, route, r.Scheme)`
- Import `gatewayv1 "sigs.k8s.io/gateway-api/apis/v1"`
- ParentRefs: single parent ref pointing to the Gateway:
  - Name: `gatewayv1.ObjectName(adminCfg.GatewayName)`
  - Namespace: `&gatewayv1.Namespace(adminCfg.GatewayNamespace)` (pointer)
  - SectionName: `&gatewayv1.SectionName("http")` (pointer)
- Hostnames: `[]gatewayv1.Hostname{gatewayv1.Hostname(hostname)}`
- Rules: single rule with single BackendRef:
  - BackendRef name: `gatewayv1.ObjectName(gs.Name)` (points to the ClusterIP Service)
  - BackendRef port: use the first port from gs.Spec.Ports if available, otherwise 8080. Use `ptrTo(gatewayv1.PortNumber(port))` (reuse ptrTo from gameserver_controller.go or define locally)
- Set annotations: `"external-dns.alpha.kubernetes.io/ttl": "60"` for fast DNS propagation
- Return error

**updateConnectionInfo method:**
- Re-fetch GameServer (fresh copy to avoid conflicts -- established pattern)
- Set `fresh.Status.Address = dnsName`
- Map ports from spec to GameServerStatusPort (same as the existing updateReadyStatus in gameserver_controller.go, but the DNS controller adds NodePort if applicable later)
- Call `r.Status().Update(ctx, fresh)`
- Record event: `r.Recorder.Eventf(gs, corev1.EventTypeNormal, "DNSConfigured", "DNS name configured: %s", dnsName)`

**SetupWithManager method:**
```go
func (r *DNSReconciler) SetupWithManager(mgr ctrl.Manager) error {
    return ctrl.NewControllerManagedBy(mgr).
        For(&gamev1alpha1.GameServer{}).
        Owns(&corev1.Service{}).
        Owns(&gatewayv1.HTTPRoute{}).
        WithEventFilter(predicate.Or(
            predicate.GenerationChangedPredicate{},
            predicate.AnnotationChangedPredicate{},
        )).
        Named("dns").
        Complete(r)
}
```
