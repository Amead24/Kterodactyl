  <action>
Create `internal/api/middleware_metrics.go` with a chi-compatible metrics middleware.

Implement a `statusRecorder` struct that wraps `http.ResponseWriter` to capture the status code:
- Embed `http.ResponseWriter`
- Add `statusCode int` field, defaulting to `http.StatusOK` (handles cases where WriteHeader is never called)
- Override `WriteHeader(code int)` to capture the status code before delegating

Implement `metricsMiddleware(next http.Handler) http.Handler` that:
1. Increments `metrics.HTTPRequestsInFlight` on entry, decrements via defer
2. Records `start := time.Now()` before calling `next.ServeHTTP(rec, r)`
3. After ServeHTTP returns, extracts the chi route pattern via `chi.RouteContext(r.Context()).RoutePattern()` — this gives low-cardinality patterns like `/api/v1/gameservers/{name}` instead of raw URLs
4. Falls back to `"unknown"` if route pattern is empty
5. Records to `metrics.HTTPRequestsTotal.WithLabelValues(method, routePattern, statusCode).Inc()`
6. Records to `metrics.HTTPRequestDuration.WithLabelValues(method, routePattern, statusCode).Observe(duration)`

Important: Do NOT implement `http.Flusher`, `http.Hijacker`, or `http.Pusher` on statusRecorder. The WebSocket console route is already mounted OUTSIDE the REST route group (line 68 of routes.go), so this middleware will never see WebSocket upgrades. Keeping the wrapper simple avoids unnecessary complexity.

Import `"github.com/kterodactyl/kterodactyl/internal/metrics"` for metric access.

**routes.go changes:**

Add `r.Use(metricsMiddleware)` inside the REST API route group (the `r.Route("/api/v1", func(r chi.Router) {` block), BEFORE the timeout and auth middleware. This ensures:
- All REST API requests are measured (authenticated and rate-limited endpoints)
- WebSocket console route (mounted at line 68, outside this group) is NOT measured
- Health check routes (/healthz, /readyz) are NOT measured (they're outside the group)
- Public auth routes (login, register) are NOT measured by this middleware (they're outside the /api/v1 group) — this is acceptable since those routes have their own rate limiting

Place `r.Use(metricsMiddleware)` as the FIRST middleware in the /api/v1 group, before `middleware.Timeout` and `s.authMiddleware.Authenticate`. This captures the full request duration including auth and timeout overhead.

No new imports needed — `metricsMiddleware` is in the same `api` package.
