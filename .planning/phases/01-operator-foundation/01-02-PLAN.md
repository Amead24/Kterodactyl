  <files>

  <action>
    Replace the scaffolded gameserver_controller.go with a full reconciliation implementation. The controller must:

    **Struct and RBAC:**
    - GameServerReconciler struct with: client.Client, Scheme *runtime.Scheme, Recorder record.EventRecorder
    - RBAC markers (from research): gameservers (full CRUD), gameservers/status (get/update/patch), gameservers/finalizers (update), pods (full CRUD), namespaces (get/list/watch/create/update/patch), resourcequotas (get/list/watch/create/update/patch), networkpolicies (get/list/watch/create/update/patch), events (create/patch)

    **Reconcile method:**
    1. Fetch GameServer CR; return nil for NotFound (deleted between queue and reconcile)
    2. Handle deletion: if DeletionTimestamp is set, call handleDeletion (remove finalizer after cleanup)
    3. Add finalizer if not present (using controllerutil.ContainsFinalizer/AddFinalizer)
    4. If Status.State is empty string, set it to GameServerStateCreating and update status, then requeue
    5. Switch on current state and call appropriate reconcile function

    **State reconciliation functions:**

    reconcileCreating(ctx, gs):
    - Extract owner from label `kterodactyl.io/owner`; if missing, set state to Error with reason "MissingOwnerLabel"
    - Call reconcilePod to create/update the Pod using controllerutil.CreateOrUpdate
    - On success, transition state to Starting
    - Update status conditions (Progressing=True, Ready=False)
    - Return requeue after 5 seconds (to check Pod status)

    reconcileStarting(ctx, gs):
    - Fetch the owned Pod by name (same name as GameServer, same namespace)
    - If Pod not found, transition back to Creating (Pod was deleted externally)
    - If Pod phase is Running AND all containers ready, transition to Ready
    - If Pod phase is Failed or Pod has been in Starting for >5 minutes, transition to Error
    - Update status conditions accordingly
    - If still Starting, requeue after 5 seconds

    reconcileReady(ctx, gs):
    - Verify Pod still exists and is Running; if not, transition to Error
    - Check for allocation annotation `kterodactyl.io/allocated: "true"` on the GameServer
    - If annotation present, transition to Allocated
    - Update status with Pod IP as address and mapped ports
    - Set condition Ready=True, Progressing=False
    - No requeue needed (watch events will trigger)

    reconcileAllocated(ctx, gs):
    - Verify Pod still exists and is Running; if not, transition to Error
    - Check if allocation annotation removed; if so, transition back to Ready
    - Check if spec indicates shutdown desired; if so, transition to Shutdown
    - No requeue needed

    reconcileShutdown(ctx, gs):
    - Delete the owned Pod if it exists (use client.Delete with GracePeriodSeconds 30)
    - Set condition Ready=False with reason "ShuttingDown"
    - Once Pod is confirmed gone, the finalizer removal in handleDeletion will complete
    - No requeue; the GameServer CR stays in Shutdown state until user deletes it

    reconcileError(ctx, gs):
    - Log the error state
    - Set condition Degraded=True
    - No requeue (manual intervention required; user can delete and recreate)

    **reconcilePod helper:**
    - Use controllerutil.CreateOrUpdate with the Pod object
    - Pod name = GameServer name, Pod namespace = GameServer namespace
    - Labels from util.GameServerLabels(owner, gameType)
    - Single container named "gameserver" with gs.Spec.Image
    - Container ports from gs.Spec.Ports
    - Container resources from gs.Spec.Resources
    - Environment variables from gs.Spec.Parameters (each key=value pair becomes an env var)
    - RestartPolicy = Never (operator manages restarts, not kubelet)
    - Set owner reference via ctrl.SetControllerReference

    **handleDeletion:**
    - Delete owned Pod if it still exists (ignore NotFound)
    - Remove finalizer
    - Update the GameServer CR

    **Status update helper:**
    - Always re-fetch the GameServer before status update to avoid conflicts (research pitfall 4)
    - Use meta.SetStatusCondition for condition updates
    - Record events using r.Recorder.Event for state transitions

    **SetupWithManager:**
    - For(&GameServer{})
    - Owns(&corev1.Pod{})
    - WithEventFilter using predicate.Or of GenerationChangedPredicate and delete predicate
    - WithOptions setting MaxConcurrentReconciles to 3
