Create `internal/api/handlers_admin.go` (replacing placeholder stubs from Plan 01):

**handleCreateInvite(w, r):**
- Decode CreateInviteRequest, validate (email required)
- Load AdminConfig per-request: cfg, err := s.loadAdminConfig(r.Context())
- Get inviting admin's username: username := usernameFromContext(r)
- Call s.inviteService.CreateInvite(r.Context(), req.Email, username, cfg.InviteExpirationHours)
- On error: respond 500 "failed to create invite"
- Respond 201 with invite details: {"token": invite.Token, "email": invite.Email, "expiresAt": invite.ExpiresAt}
- Note: Token is returned so admin can share the link manually if SMTP is not configured

**handleListUsers(w, r):**
- Call s.userStore.ListUsers(r.Context())
- On error: respond 500 "failed to list users"
- Map to safe response (omit PasswordHash!):
  - UserResponse struct: Username, Email, Role, CreatedAt, InvitedBy
- respondList(w, 200, responses, len(responses))

Define `UserResponse` struct: Username string, Email string, Role string, CreatedAt string, InvitedBy string. All json tagged.

Define `userToResponse(u *auth.User) *UserResponse` mapping (explicitly exclude PasswordHash).

**handleDeleteUser(w, r):**
- username := chi.URLParam(r, "username")
- If username is empty: respond 400 "username is required"
- Prevent self-deletion: adminUsername := usernameFromContext(r); if adminUsername == username: respond 400 "cannot delete yourself"
- Call s.userStore.DeleteUser(r.Context(), username)
- On auth.ErrUserNotFound: respond 404 "user not found"
- On other error: respond 500 "failed to delete user"
- Respond 204 No Content

Create `internal/api/handlers_admin_test.go`:

**TestHandleCreateInvite:**

**TestHandleListUsers:**

**TestHandleDeleteUser:**
- admin deletes existing user -> 204
- admin deletes non-existent user -> 404
- admin tries to delete self -> 400
- non-admin -> 403

For admin tests, create JWT tokens with role="admin" and role="user" to verify RequireAdmin enforcement.

For the fake K8s client in admin tests, the InviteService needs a client that can create Secrets (invite Secrets). Use fake.NewClientBuilder() with corev1 scheme registered. Pre-populate the fake client with user Secrets for list/delete tests.

Since the fake client won't have the ConfigMap, defaults will be used. This is acceptable for tests.
