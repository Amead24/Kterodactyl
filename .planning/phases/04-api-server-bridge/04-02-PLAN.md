Create `internal/api/handlers_auth.go` (replacing placeholder stubs from Plan 01):

**handleLogin(w, r):**
- Decode LoginRequest, validate (username and password required)
- Call s.userStore.GetUser(ctx, req.Username)
- On ErrUserNotFound: respond 401 "invalid credentials" (don't reveal which field is wrong)
- Call auth.VerifyPassword(req.Password, user.PasswordHash)
- On failure: respond 401 "invalid credentials"
- Call s.jwtService.GenerateToken(user)
- On error: respond 500 "failed to generate token"
- Respond 200 with {"token": "..."}

**handleRegister(w, r):**
- Decode RegisterRequest, validate (username, email, password, inviteToken all required)
- Load AdminConfig per-request via s.loadAdminConfig(ctx) -- check RegistrationEnabled flag
- If registration disabled: respond 403 "registration is disabled"
- Call s.inviteService.RedeemInvite(ctx, req.InviteToken)
- On ErrInvalidToken: respond 400 "invalid invite token"
- On ErrInviteExpired: respond 400 "invite token has expired"
- Validate username via auth.ValidateUsername(req.Username)
- On ErrInvalidUsername: respond 400 with the wrapped message
- Hash password via auth.HashPassword(req.Password)
- Create user via s.userStore.CreateUser with Role: auth.RoleUser, Email from invite redeem result (use invite email, not request email -- the invite is for a specific email)
- On ErrUserExists: respond 409 "username already taken"
- Generate JWT token for the new user
- Respond 201 with {"token": "...", "username": "..."}

**handleRefresh(w, r):**
- Get claims from context via auth.GetUserFromContext
- Reconstruct User from claims: Username, Email, Role
- Call s.jwtService.GenerateToken(user)
- Respond 200 with {"token": "..."}

Create `internal/api/handlers_gameserver.go` (replacing placeholder stubs):

Define `GameServerResponse` struct for API responses (do NOT expose raw K8s objects -- Pitfall: never expose K8s API directly):
- Name string `json:"name"`
- GameType string `json:"gameType"`
- State string `json:"state"`
- Address string `json:"address,omitempty"`
- Ports []PortResponse `json:"ports,omitempty"`
- Parameters map[string]string `json:"parameters,omitempty"`
- CreatedAt string `json:"createdAt"`

Define `PortResponse` struct: Name string, Port int32, Protocol string.

Define `gameServerToResponse(gs *gamev1alpha1.GameServer) *GameServerResponse` that maps CRD fields to response struct. Map status.ports to PortResponse, use metadata.creationTimestamp for createdAt (format RFC3339).

**handleListGameServers(w, r):**
- ns := namespaceFromContext(r) -- ALWAYS from JWT claims (Pitfall 1)
- list := &gamev1alpha1.GameServerList{}
- s.client.List(ctx, list, client.InNamespace(ns))
- Map items to []GameServerResponse
- respondList(w, 200, responses, len(responses))

**handleCreateGameServer(w, r):**
- ns := namespaceFromContext(r)
- username := usernameFromContext(r)
- Decode CreateGameServerRequest, validate
- Look up manifest: s.manifestLoader.Get(req.GameType) -- if not found, respond 400 "unknown game type"
- Build GameServer CR:
  - Name: req.Name, Namespace: ns
  - Labels: util.GameServerLabels(username, req.GameType) -- Pitfall 7: always use this
  - Spec: GameType from request, Image from manifest, Ports from manifest, Resources from manifest, Parameters merged (manifest defaults + user overrides via mergeMaps helper)
- s.client.Create(ctx, gs)
- Handle k8serrors.IsAlreadyExists -> respond 409 "game server with this name already exists" (Pitfall 5)
- Respond 201 with gameServerToResponse(gs) -- return spec immediately, status populates async (Pitfall 2)

**handleGetGameServer(w, r):**
- ns := namespaceFromContext(r)
- name := chi.URLParam(r, "name")
- gs := &gamev1alpha1.GameServer{}
- s.client.Get(ctx, client.ObjectKey{Name: name, Namespace: ns}, gs)
- Handle k8serrors.IsNotFound -> respond 404
- Respond 200 with gameServerToResponse(gs)

**handleUpdateGameServer(w, r):**
- ns := namespaceFromContext(r)
- name := chi.URLParam(r, "name")
- Decode UpdateGameServerRequest, validate
- Fetch existing: s.client.Get(ctx, ...)
- Handle not found -> 404
- Only update Parameters (not GameType, Image, etc. -- per research recommendation)
- Update spec.Parameters by merging existing with request
- s.client.Update(ctx, gs)
- Respond 200 with gameServerToResponse(gs)

**handleDeleteGameServer(w, r):**
- ns := namespaceFromContext(r)
- name := chi.URLParam(r, "name")
- gs := &gamev1alpha1.GameServer{}
- s.client.Get(ctx, ...)
- Handle not found -> 404
- s.client.Delete(ctx, gs)
- Respond 204 No Content (empty body)

Helper: `mergeMaps(base, override map[string]string) map[string]string` -- copies base, overlays override values.
