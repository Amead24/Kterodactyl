Create `internal/manifest/manifest.go`:

Define `GameManifest` struct with fields:
- Name string (yaml:"name") -- must match filename stem
- DisplayName string (yaml:"displayName")
- Image string (yaml:"image")
- Ports []gamev1alpha1.GameServerPort (yaml:"ports")
- Parameters map[string]string (yaml:"parameters")
- Resources corev1.ResourceRequirements (yaml:"resources")

Import `gopkg.in/yaml.v3` for parsing (already indirect dep in go.mod).
Import `github.com/kterodactyl/kterodactyl/api/v1alpha1` for GameServerPort type.
Import `k8s.io/api/core/v1` for ResourceRequirements and Protocol.

Define `Loader` struct holding `manifests map[string]*GameManifest`.

`LoadFromDirectory(dir string) (*Loader, error)`:
- Read all .yaml/.yml files from dir using os.ReadDir
- For each file: read contents, yaml.Unmarshal into GameManifest
- Validate: Name must be non-empty, Image must be non-empty
- Store in map keyed by manifest.Name
- Return error if dir doesn't exist or no manifests found

`(l *Loader) Get(name string) (*GameManifest, bool)`:
- Lookup by name in map, return (manifest, found)

`(l *Loader) List() []*GameManifest`:
- Return all manifests sorted by Name alphabetically

  <files>internal/api/server.go, internal/api/routes.go, internal/api/response.go, internal/api/request.go, internal/api/middleware.go, internal/api/handlers_health.go</files>

Create `internal/api/response.go`:
- `ErrorResponse` struct: Error string `json:"error"`, Details string `json:"details,omitempty"`
- `respondJSON(w http.ResponseWriter, status int, data interface{})`: sets Content-Type application/json, writes status, encodes data
- `respondError(w http.ResponseWriter, status int, message string)`: calls respondJSON with ErrorResponse
- `respondList(w http.ResponseWriter, status int, data interface{}, count int)`: responds with `{"data": ..., "count": N}`

Create `internal/api/request.go`:
- `decodeJSON(r *http.Request, v interface{}) error`: uses json.NewDecoder with DisallowUnknownFields=false (accept extra fields gracefully), decodes, returns error
- `CreateGameServerRequest` struct: Name string `json:"name"`, GameType string `json:"gameType"`, Parameters map[string]string `json:"parameters,omitempty"`
- `LoginRequest` struct: Username string `json:"username"`, Password string `json:"password"`
- `RegisterRequest` struct: Username string `json:"username"`, Email string `json:"email"`, Password string `json:"password"`, InviteToken string `json:"inviteToken"`
- `CreateInviteRequest` struct: Email string `json:"email"`
- `UpdateGameServerRequest` struct: Parameters map[string]string `json:"parameters,omitempty"`
- Each request type gets a `Validate() error` method that checks required fields and returns descriptive error messages

Create `internal/api/middleware.go`:
- `namespaceFromContext(r *http.Request) string`: extracts claims via auth.GetUserFromContext(r.Context()), returns claims.Namespace. Returns "" if no claims (middleware should prevent this).
- `usernameFromContext(r *http.Request) string`: extracts claims, returns claims.Username.
- `(s *Server) loadAdminConfig(ctx context.Context) (*controller.AdminConfig, error)`: loads AdminConfig per-request using controller.LoadAdminConfig(ctx, s.client, s.operatorNamespace). This avoids staleness (Pitfall 3).

Create `internal/api/handlers_health.go`:
- `handleHealthz(w http.ResponseWriter, r *http.Request)`: responds 200 with `{"status":"ok"}`
- `handleReadyz(w http.ResponseWriter, r *http.Request)`: responds 200 with `{"status":"ok"}` (can be extended later with dependency checks)

Create `internal/api/server.go`:
- `Config` struct with fields: Client client.Client, JWTService *auth.JWTService, UserStore auth.UserService, InviteService *auth.InviteService, ManifestLoader *manifest.Loader, OperatorNamespace string, BindAddress string
- `Server` struct with private fields mirroring Config plus: authMiddleware *auth.AuthMiddleware, router chi.Router
- `NewServer(cfg Config) *Server`: constructs Server, creates authMiddleware via auth.NewAuthMiddleware(cfg.JWTService), calls s.routes() to build router
- `HTTPServer() *http.Server`: returns &http.Server{Addr: s.bindAddress, Handler: s.router, ReadHeaderTimeout: 10*time.Second, IdleTimeout: 120*time.Second}

Create `internal/api/routes.go`:
- `(s *Server) routes() chi.Router`: builds and returns the chi router
- Global middleware (in this order): middleware.RequestID, middleware.RealIP, middleware.Logger, middleware.Recoverer, middleware.Timeout(30*time.Second)
- CORS at top level (Pitfall 4): cors.Handler with AllowedOrigins: []string{"*"}, AllowedMethods: GET/POST/PUT/DELETE/OPTIONS, AllowedHeaders: Authorization/Content-Type, ExposedHeaders: X-Refresh-Token, AllowCredentials: true, MaxAge: 300
- Global rate limit: httprate.LimitByIP(100, time.Minute)
- Health routes (unauthenticated): GET /healthz, GET /readyz
- Public auth routes with tighter rate limits:
  - POST /api/v1/auth/login (rate limited: 5/min via httprate.LimitByIP inline)
  - POST /api/v1/auth/register (rate limited: 3/min via httprate.LimitByIP inline)
- Authenticated group /api/v1 with s.authMiddleware.Authenticate:
  - POST /auth/refresh -> s.handleRefresh
  - GET /games -> s.handleListGames
  - GET /games/{gameType} -> s.handleGetGame
  - Route group /gameservers with create rate limit (10/min):
    - GET / -> s.handleListGameServers
    - POST / -> s.handleCreateGameServer (the rate-limited one)
    - Route group /{name}: GET /, PUT /, DELETE /
  - Admin route group /admin with auth.RequireAdmin:
    - POST /invites -> s.handleCreateInvite
    - GET /users -> s.handleListUsers
    - DELETE /users/{username} -> s.handleDeleteUser

For now, create placeholder handler methods on Server for handlers not yet implemented (Plans 02/03). Each placeholder responds 501 Not Implemented. This lets the router compile and lets us verify health endpoints work.

Placeholder stubs needed: handleLogin, handleRegister, handleRefresh, handleListGames, handleGetGame, handleListGameServers, handleCreateGameServer, handleGetGameServer, handleUpdateGameServer, handleDeleteGameServer, handleCreateInvite, handleListUsers, handleDeleteUser.
