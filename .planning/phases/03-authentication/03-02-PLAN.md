  <name>Task 1: Create JWT service with signing key persistence</name>
  <files>internal/auth/jwt.go</files>
  <action>
  First, add the golang-jwt dependency:
  ```
  go get github.com/golang-jwt/jwt/v5@v5.3.1
  ```

  Create `internal/auth/jwt.go`:
  - Package `auth`
  - Import `github.com/golang-jwt/jwt/v5`, `sigs.k8s.io/controller-runtime/pkg/client`, `internal/util`
  - Define `const jwtKeySecretName = "kterodactyl-jwt-signing-key"`

  Define `KterodactylClaims` struct:
  - Embed `jwt.RegisteredClaims`
  - Fields: Username, Email, Role, Namespace (all string with json tags)

  Define `JWTService` struct:
  - `signingKey []byte`
  - `tokenExpiration time.Duration`
  - `refreshThreshold time.Duration` (how close to expiry before auto-refresh; default 2 hours)

  Constructor:
  - `NewJWTService(signingKey []byte, expiration time.Duration) *JWTService`
  - Set refreshThreshold to 2 hours

  Methods:
  - `GenerateToken(user *User) (string, error)`:
    - Create `KterodactylClaims` with Subject=username, Issuer="kterodactyl", IssuedAt=now, ExpiresAt=now+expiration
    - Set Username, Email, Role from user
    - Set Namespace to `util.UserNamespace(user.Username)`
    - Generate unique token ID via `generateTokenID()` (8 random bytes hex-encoded)
    - Sign with HMAC-SHA256 (`jwt.SigningMethodHS256`)
  - `ValidateToken(tokenString string) (*KterodactylClaims, error)`:
    - Parse with `jwt.ParseWithClaims` using `KterodactylClaims`
    - Use key function returning `s.signingKey`
    - Validation options: `jwt.WithValidMethods([]string{"HS256"})`, `jwt.WithIssuer("kterodactyl")`, `jwt.WithExpirationRequired()`
    - Return claims on success, wrapped error on failure
  - `ShouldRefresh(claims *KterodactylClaims) bool`:
    - Returns true if token expires within `s.refreshThreshold`
    - Used by middleware to decide if a refreshed token should be issued

  Static function:
  - `EnsureSigningKey(ctx context.Context, c client.Client, namespace string) ([]byte, error)`:
    - Try to Get Secret `kterodactyl-jwt-signing-key` from namespace
    - If NotFound: generate 32 random bytes via `crypto/rand`, create Secret with labels (`kterodactyl.io/managed-by: kterodactyl`, `kterodactyl.io/resource-type: jwt-key`), return the key
    - If found: return `secret.Data["signing-key"]`
    - Return error on any other failure

  Helper:
  - `generateTokenID() string` -- 8 bytes from crypto/rand, hex-encoded to 16 chars

  Follow the exact patterns from research document's "JWT Token Generation and Validation" and "JWT Signing Key Storage" sections.
  </action>
  <verify>
  ```
  cd /home/tony/kterodactyl && go build ./internal/auth/...
  ```
  Confirm JWT service compiles. Verify KterodactylClaims and JWTService are exported.
  </verify>
  <done>
  - `JWTService` generates and validates HMAC-SHA256 JWT tokens
  - `KterodactylClaims` includes Username, Email, Role, Namespace
  - `EnsureSigningKey` creates or loads signing key from K8s Secret
  - `ShouldRefresh` detects tokens nearing expiry for auto-refresh
  - Token validation enforces HS256 method, "kterodactyl" issuer, and expiration
