  <name>Task 1: Create auth types, errors, and password hashing</name>
  <files>internal/auth/auth.go, internal/auth/errors.go, internal/auth/password.go</files>
  <action>
  First, add the golang.org/x/crypto dependency:
  ```
  go get golang.org/x/crypto
  ```

  Create `internal/auth/errors.go`:
  - Package `auth`
  - Define sentinel errors using `errors.New()`:
    - `ErrUserExists` - "user already exists"
    - `ErrUserNotFound` - "user not found"
    - `ErrInvalidCredentials` - "invalid credentials"
    - `ErrInvalidToken` - "invalid token"
    - `ErrTokenExpired` - "token expired"
    - `ErrInviteExpired` - "invitation expired"
    - `ErrInviteUsed` - "invitation already used"
    - `ErrInvalidUsername` - "invalid username"
    - `ErrRegistrationDisabled` - "registration is disabled"

  Create `internal/auth/auth.go`:
  - Package `auth`
  - Define `User` struct: Username, Email, PasswordHash, Role, CreatedAt, InvitedBy (all strings)
  - Define role constants: `RoleAdmin = "admin"`, `RoleUser = "user"`
  - Define `UserService` interface with methods: CreateUser, GetUser, GetUserByEmail, ListUsers, DeleteUser (all take context.Context)
  - Define `ValidateUsername(username string) error` function:
    - Must be 1-63 characters
    - Must match DNS label regex: `^[a-z0-9]([a-z0-9-]*[a-z0-9])?$`
    - Must not be a reserved name (list: "admin", "system", "operator", "default", "kube")
    - Returns `ErrInvalidUsername` with descriptive wrapping on failure

  Create `internal/auth/password.go`:
  - Package `auth`
  - Implement Argon2id hashing using `golang.org/x/crypto/argon2` with OWASP-recommended parameters:
    - time=1, memory=64*1024 (64 MB), threads=4, keyLen=32, saltLen=16
  - `HashPassword(password string) (string, error)`:
    - Generate 16-byte salt via `crypto/rand`
    - Hash with `argon2.IDKey()`
    - Return PHC string format: `$argon2id$v=19$m=65536,t=1,p=4$<base64-salt>$<base64-hash>`
  - `VerifyPassword(password, encodedHash string) (bool, error)`:
    - Parse PHC string format (split on `$`, expect 6 parts)
    - Extract version, memory, time, threads from parts[2] and parts[3]
    - Decode salt and expected hash from base64
    - Re-derive hash with same parameters
    - Compare using `crypto/subtle.ConstantTimeCompare()`
  - Follow the exact code patterns from the research document's "Argon2id Password Hashing" section
  </action>

  Confirm no compilation errors. Verify the package exists and exports the expected types.
  </verify>

  - `auth.User` struct defined with all fields
  - `auth.ValidateUsername()` rejects invalid DNS labels and reserved names
  - `auth.HashPassword()` returns PHC-format Argon2id hash
  - `auth.VerifyPassword()` correctly validates passwords with constant-time comparison

    - Return `ErrUserExists` if `errors.IsAlreadyExists(err)`
