**4. Create `e2e/fixtures/auth.ts`:**
Custom `test.extend()` that provides `adminPage` and `userPage` fixtures. Each fixture:
- Reads the JWT token from `playwright/.auth/admin.json` or `playwright/.auth/user.json`
- Creates a new browser context
- Calls `context.addInitScript()` to set `window.__KTERODACTYL_E2E_TOKEN = token`
- Creates a page from that context
- Yields the page via `use(page)`
- Closes context after test

Export `test` (extended) and re-export `expect` from `@playwright/test`.

**5. Create `e2e/tests/auth.setup.ts`:**
This setup project runs before all chromium tests:

Step A — Seed admin user via kubectl:
- Use `execSync` from `child_process`
- Constants: `ADMIN_USERNAME = 'e2e-admin'`, `ADMIN_PASSWORD = 'e2e-test-password'`, `ADMIN_EMAIL = 'admin@e2e-test.local'`, `NAMESPACE = 'kterodactyl-system'`
- Generate the Argon2id password hash by running a small Go program inline: `go run -e 'package main; import ("fmt"; "github.com/kterodactyl/kterodactyl/internal/auth"); func main() { h, _ := auth.HashPassword("e2e-test-password"); fmt.Print(h) }'`. Actually, simpler approach: create a tiny Go script `hack/hash-password.go` that imports the project's auth package. OR even simpler: use `execSync` to run `go run ./hack/hash-password.go "e2e-test-password"` from the project root.

  **Best approach:** Create `hack/hash-password.go` — a small main program that takes a password arg and prints the Argon2id hash. Then in the setup, call `execSync('go run ./hack/hash-password.go e2e-test-password', { cwd: projectRoot })` to get the hash. This avoids hardcoding hashes and uses the project's actual hash implementation.

  If the `go run` approach is not viable in CI (no Go), pre-compute the hash and store as a constant. But Go IS available in the kind CI environment.

- Create the kubectl secret: `kubectl create secret generic user-e2e-admin --namespace=kterodactyl-system --from-literal=email=admin@e2e-test.local --from-literal=password-hash='<hash>' --from-literal=created-at=2026-01-01T00:00:00Z --from-literal=invited-by=bootstrap --dry-run=client -o yaml | kubectl apply -f -`
- Label the secret: `kubectl label secret user-e2e-admin --namespace=kterodactyl-system --overwrite kterodactyl.io/resource-type=user kterodactyl.io/user=e2e-admin kterodactyl.io/role=admin kterodactyl.io/managed-by=kterodactyl`

Step B — Log in as admin via API:
- POST `http://localhost:8080/api/v1/auth/login` with `{ username: 'e2e-admin', password: 'e2e-test-password' }`
- Extract `{ token }` from response
- Write `{ token }` to `playwright/.auth/admin.json` (mkdir -p first)

Step C — Create invite and register regular user:
- POST `http://localhost:8080/api/v1/admin/invites` with `Authorization: Bearer <admin-token>` and body `{ email: 'e2e-user@test.local' }`
- Extract `{ token: inviteToken }` from response
- POST `http://localhost:8080/api/v1/auth/register` with `{ username: 'e2e-user', email: 'e2e-user@test.local', password: 'e2e-test-password', inviteToken }`
- Extract `{ token }` from response
- Write `{ token }` to `playwright/.auth/user.json`

Use Playwright's `request` context (from `test` fixtures) for all API calls.

**6. Create `hack/hash-password.go`:**
```go
//go:build ignore

package main

import (
    "fmt"
    "os"
    "github.com/kterodactyl/kterodactyl/internal/auth"
)

func main() {
    if len(os.Args) < 2 {
        fmt.Fprintln(os.Stderr, "usage: hash-password <password>")
        os.Exit(1)
    }
    hash, err := auth.HashPassword(os.Args[1])
    if err != nil {
        fmt.Fprintln(os.Stderr, err)
        os.Exit(1)
    }
    fmt.Print(hash)
}
```

Add this block AFTER the `create()` call (before the final export or at the end of the file):

```typescript
// E2E test support: if a token was injected via Playwright's addInitScript(),

if (typeof window !== 'undefined' && (window as any).__KTERODACTYL_E2E_TOKEN) {

}
```
