**1. Create `internal/api/handlers_backups.go`:**

Define response/request types:
```go
type BackupResponse struct {
    Name           string  `json:"name"`
    GameServerName string  `json:"gameServerName"`
    State          string  `json:"state"`
    S3Key          string  `json:"s3Key,omitempty"`
    S3Bucket       string  `json:"s3Bucket,omitempty"`
    Size           int64   `json:"size,omitempty"`
    StartedAt      string  `json:"startedAt,omitempty"`
    CompletedAt    string  `json:"completedAt,omitempty"`
    Message        string  `json:"message,omitempty"`
    CreatedAt      string  `json:"createdAt"`
}

type SetBackupScheduleRequest struct {
    Schedule  string `json:"schedule"`  // Cron expression, empty to disable
    Retention int    `json:"retention"` // Max backups to retain (0 = use admin default)
}
```

Create `backupToResponse(backup *gamev1alpha1.Backup) BackupResponse` helper that maps CRD fields to API response (same pattern as `gameServerToResponse`). Format times as RFC3339.

**handleCreateBackup** (POST /api/v1/gameservers/{name}/backups):
1. Extract namespace from JWT context, server name from URL param
2. Fetch the GameServer. Return 404 if not found
3. Verify GameServer state is Ready or Allocated. Return 409 if not
4. Check for existing InProgress backup (list Backups with label `kterodactyl.io/backup-gameserver=<name>`). Return 409 "backup already in progress" if found
5. Create a Backup CR in the GameServer's namespace:
   - Name: `<servername>-backup-<timestamp>` where timestamp is `time.Now().UTC().Format("20060102-150405")`
   - Labels: `LabelBackupGameServer: serverName`, `LabelManagedBy: ManagedByValue`
   - Spec.GameServerName: serverName
   - Status.State: BackupStatePending
6. Use `Status().Update()` to set initial state after creation (since status is a subresource)
7. Return 201 with BackupResponse

**handleListBackups** (GET /api/v1/gameservers/{name}/backups):
1. Extract namespace from JWT context, server name from URL param
2. Fetch the GameServer. Return 404 if not found (ownership check)
3. List Backup CRs in namespace with label selector `kterodactyl.io/backup-gameserver=<name>`
4. Sort by creation time (newest first)
5. Map to BackupResponse slice
6. Return via `respondList()`

**handleDeleteBackup** (DELETE /api/v1/gameservers/{name}/backups/{backupName}):
- Admin-only endpoint (RequireAdmin middleware applied in routes)
1. Extract namespace from JWT context, server name and backup name from URL params
2. Fetch the Backup CR. Return 404 if not found
3. Verify the Backup's GameServerName matches the server name from URL (prevent cross-server access)
4. Do NOT delete the S3 object here -- the BackupReconciler's finalizer (or a separate cleanup) handles S3 deletion. For v1 simplicity, just delete the CR and note that S3 cleanup is a future enhancement (orphan S3 objects are acceptable for homelab v1)
5. Delete the Backup CR
6. Return 204 No Content

**handleRestoreBackup** (POST /api/v1/gameservers/{name}/backups/{backupName}/restore):
- Admin-only endpoint
1. Extract namespace from JWT context, server name and backup name from URL params
2. Fetch the GameServer. Return 404 if not found
3. Fetch the Backup CR. Return 404 if not found
4. Verify Backup state is Completed. Return 409 if not
5. Verify GameServer state is Ready or Allocated. Return 409 if not running
6. Load AdminConfig for S3 settings. If S3 not configured, return 503
7. Create S3 client (same pattern as BackupReconciler's ensureS3Client but on the API server side -- since the API server also has access to the K8s client to read Secrets)
8. Perform restore: S3 GetObject -> gzip.NewReader -> exec `tar -xf - -C <backupPath>` in the pod (reuse the execInPod helper from handlers_mods.go)
9. After successful restore, restart the server (set state to Creating via Status().Update())
10. Return 200 with a message: `{"message": "restored from backup <name>, server restarting"}`

Note: The restore could also be done by having the API create a "restore" annotation on the Backup CR and letting the BackupReconciler handle it. However, for v1, doing it directly in the API handler is simpler and follows the same pattern as mod upload (direct exec from API server).

**handleSetBackupSchedule** (PUT /api/v1/gameservers/{name}/backup-schedule):
- Admin-only endpoint
1. Decode SetBackupScheduleRequest from body
2. Fetch the GameServer. Return 404 if not found
3. If schedule is empty: remove schedule/retention annotations from GameServer
4. If schedule is non-empty: validate cron expression using `robfig/cron/v3` parser. Return 400 if invalid
5. Set annotations on GameServer: AnnotationBackupSchedule, AnnotationBackupRetention (if > 0)
6. Update the GameServer
7. Return 200 with updated schedule info

**Update `internal/api/routes.go`:**

Add backup routes inside the existing `/{name}` route group under `/gameservers`, after the `/mods` route group:

```go
// Backup management
r.Route("/backups", func(r chi.Router) {
    r.Post("/", s.handleCreateBackup)
    r.Get("/", s.handleListBackups)
    r.Route("/{backupName}", func(r chi.Router) {
        r.Use(auth.RequireAdmin) // Delete and restore are admin-only
        r.Delete("/", s.handleDeleteBackup)
        r.Post("/restore", s.handleRestoreBackup)
    })
})
```

Add the schedule endpoint (admin-only) inside the `/{name}` route group:
```go
// Backup schedule (admin-only)
r.With(auth.RequireAdmin).Put("/backup-schedule", s.handleSetBackupSchedule)
```

The route tree for backups will look like:
```
/api/v1/gameservers/{name}/backups          POST   (create backup - any user)
/api/v1/gameservers/{name}/backups          GET    (list backups - any user)
/api/v1/gameservers/{name}/backups/{backupName}      DELETE (admin only)
/api/v1/gameservers/{name}/backups/{backupName}/restore  POST (admin only)
/api/v1/gameservers/{name}/backup-schedule  PUT    (admin only)
```

Ensure the `auth` import is present (it should already be imported for `auth.RequireAdmin`).

NOTE: Keep the existing comment about timeout applying to REST routes. Backup/restore operations use the 30s timeout from the route group. For v1 homelab with small game servers, this is sufficient. The BackupReconciler handles the heavy lifting asynchronously -- the API handler only creates a Backup CR (instant) for the create endpoint. The restore endpoint does the S3 download + exec, which may need more time for large backups. Add a comment: "// NOTE: Restore may need timeout increase for large backups in v2."
