Define the reconciler struct:
```go
type BackupReconciler struct {
    client.Client
    Scheme            *runtime.Scheme
    Recorder          record.EventRecorder
    OperatorNamespace string
    Clientset         *kubernetes.Clientset
    RestConfig        *rest.Config

    // s3Client is lazily initialized on first backup
    s3Client *minio.Client
    s3Bucket string
}
```

Implement the `Reconcile` method:
1. Fetch the Backup CR. If not found, return (deleted).
2. If terminal state (Completed/Failed), skip.
3. Load AdminConfig for S3 settings.
4. Switch on `backup.Status.State`:
   - **Empty/Pending**: Validate S3 is configured (BackupS3Endpoint non-empty). If not, transition to Failed with message "S3 not configured". Otherwise, transition to InProgress, set StartedAt, requeue immediately.
   - **InProgress**: Perform the actual backup:
     a. Fetch the referenced GameServer. If not found, fail.
     b. Check GameServer state is Ready or Allocated. If not, fail with "server must be running".
     c. Check no other InProgress backup exists for this GameServer (list Backups by label).
     d. Get backupPath from GameServer annotation (AnnotationBackupPath). Default to "/data" if empty.
     e. Initialize S3 client lazily (call `ensureS3Client`).
     f. Auto-create bucket if not exists (BucketExists + MakeBucket).
     g. Call `performBackup()` -- the tar->gzip->S3 pipeline.
     h. On success: set S3Key, S3Bucket, Size, CompletedAt, transition to Completed.
     i. On failure: set Message, transition to Failed.

Implement `performBackup()`:
- Use `io.Pipe` to connect exec stdout to S3 upload
- Goroutine: exec `tar -cf - -C <backupPath> .` in the game server pod, pipe stdout through `gzip.NewWriter` to the pipe writer
- Main thread: `s3Client.PutObject(ctx, bucket, s3Key, pipeReader, -1, minio.PutObjectOptions{ContentType: "application/gzip", PartSize: 64 * 1024 * 1024})`
- S3 key format: `backups/<namespace>/<gameserver-name>/<timestamp>.tar.gz` where timestamp is `time.Now().UTC().Format("20060102-150405")`
- Return the minio.UploadInfo (contains Size) and error

Implement `performRestore()`:
- `s3Client.GetObject(ctx, bucket, s3Key, minio.GetObjectOptions{})`
- Pipe through `gzip.NewReader`
- Exec `tar -xf - -C <backupPath>` in the game server pod with the gzip reader as stdin
- After restore, restart the server: set state to Creating via Status().Update()

Implement `ensureS3Client()`:
- If s3Client already set, return
- Load S3 credentials Secret named `kterodactyl-s3-credentials` from operator namespace
- Extract `accessKeyID` and `secretAccessKey` from Secret data
- Create `minio.New(endpoint, &minio.Options{Creds: credentials.NewStaticV4(accessKey, secretKey, ""), Secure: useSSL})`
- Cache on reconciler struct

Implement `execTarFromPod()` using the SPDY remotecommand pattern (similar to the execInPod in handlers_mods.go but the stdout goes to a writer instead of a buffer):
```go
func (r *BackupReconciler) execTarFromPod(ctx context.Context, namespace, podName, backupPath string, stdout io.Writer) error {
    req := r.Clientset.CoreV1().RESTClient().Post().
        Resource("pods").Name(podName).Namespace(namespace).
        SubResource("exec").
        VersionedParams(&corev1.PodExecOptions{
            Container: "gameserver",
            Command:   []string{"tar", "-cf", "-", "-C", backupPath, "."},
            Stdout:    true,
            Stderr:    true,
        }, scheme.ParameterCodec)

    exec, err := remotecommand.NewSPDYExecutor(r.RestConfig, "POST", req.URL())
    // ...stream stdout to the provided writer, capture stderr
}
```

**Actually, simplify the scheduling approach:** Rather than complicating the Backup reconciler with GameServer watches, create a separate periodic check. The BackupReconciler reconciles Backups only. For scheduling, add a simple goroutine (or a second controller) that periodically lists GameServers with schedule annotations and creates Backup CRs. However, the simplest approach for v1: **handle scheduling in the API layer** -- admin sets the schedule annotation, and a periodic requeue in the BackupReconciler checks GameServers on each pass.

**Simplified scheduling (recommended for v1):** The BackupReconciler has a Watches on GameServer that triggers reconciliation. When it sees a GameServer with a backup-schedule annotation:
1. Parse cron, check last-backup-time
2. If due: create Backup CR
3. Return RequeueAfter to next schedule time

To avoid complexity, implement scheduling as a separate `SetupWithManager` watch. When reconciling a GameServer event: only handle schedule logic (not backup execution). When reconciling a Backup event: handle the backup state machine.
